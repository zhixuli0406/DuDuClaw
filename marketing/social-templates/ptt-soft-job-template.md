# PTT Soft_Job 板文章模板

## 模板：開發心得分享文

### 標題格式
`[心得] {吸引眼球的技術描述}`

### 範例標題
- `[心得] 花了一年用 Rust 打造自我進化的 AI Agent 系統`
- `[心得] 在 Mac Mini 上跑 70B 模型的踩坑紀錄`
- `[心得] 用 12 個 Rust Crate 取代 3 套月租 SaaS`

### 文章結構

```
【前言】
（個人痛點出發，2-3 段）
- 為什麼做這個？遇到什麼問題？
- 市面上的方案為什麼不滿意？

【技術選型】
（每個選擇附理由，3-5 段）
- 為什麼選 Rust？（效能、安全、單一 binary）
- 為什麼不用 Python/Node？
- 為什麼自建 Evolution Engine？

【架構介紹】
（搭配 ASCII art 或圖片，2-3 段）
- 整體架構圖
- 關鍵模組說明
- 資料流程

【成果展示】
（具體數字，2-3 段）
- 效能數據（回應速度、記憶體用量）
- 成本數據（API 費用節省多少）
- 實際使用場景

【踩坑紀錄】
（最有價值的部分，3-5 段）
- 遇到的技術難題
- 如何解決的
- 事後回想會怎麼做

【結語】
（自然帶出專案，1-2 段）
- 專案開源在 GitHub
- 歡迎交流討論
- 不放定價、不放購買連結
```

### 注意事項
- ❌ 避免：商業推銷感、定價資訊、購買連結
- ✅ 適合：GitHub 連結、技術討論、個人心得
- 語氣：工程師之間的分享，不是業務推銷
- 每 2 週 1 篇，持續建立社群存在感
