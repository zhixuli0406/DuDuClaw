## 三、阻礙事項（Blockers）

### BLOCKER-001｜CostTelemetry 尚未啟動

**描述**：Token Cost Defense Phase 0（CostTelemetry 觀測層）尚未實作。  
**影響**：
- 無法知道各 Agent 的真實 token 消耗，cache 效率不可見
- Phase 2.5 Multi-OAuth 的「rate limit 時自動切換帳號」邏輯依賴 telemetry
- Phase 3 Subprocess Pool 的 pool hit/miss rate 追蹤依賴 telemetry
- DuDu Dreaming Worker 如果要優化 local vs cloud 路徑，也需要 cost 信號

**解法**：T2 列為今日 P0，基礎設施工程師本週全力攻克 Phase 0 的 7 個子任務。  
**預計解除時間**：2026-05-12（本週五）

---

### BLOCKER-002｜DuDu Dreaming 缺乏 Local Inference 後端

**描述**：DuDu Dreaming 設計時預期使用本地 LLM（MLX / llama.cpp）執行 session 掃描，以降低成本。但 `duduclaw-inference` crate 尚未建立。

**影響**：Dreaming Worker 若只走 cloud API，成本不可控。

**解法（TL 技術裁定）**：
> DuDu Dreaming Phase 1 先用 Confidence Router（已有）做本地 fallback，Inference crate 獨立為 P2 任務，不阻塞 Dreaming SDD。Dreaming 設計文件中需標記「Local Inference 整合點」，待 crate 完成後 plug-in。

---

### BLOCKER-003｜Anthropic Dreaming 技術細節不明

**描述**：PM 已蒐集競品方向，但 Anthropic Dreaming 的內部實作（觸發條件、去重算法、memory store 格式）尚不清楚，可能影響 DuDuClaw SDD 設計決策。

**解法**：記憶系統工程師在撰寫 SDD 前，需先做半天的深度競品拆解（結合 AgeMem arXiv:2601.01885 + Memori arXiv:2603.19935），確保設計有差異化。

## 四、技術決策

### TL-DECISION-2026-05-08-001：A2A Protocol 實作範圍

**議題**：PM 提案建議完整實作 Google A2A 標準（AgentCard + Task + Message）。

**TL 裁定**：**Phase 1 只做 Transport Layer**，原因：
1. Google A2A spec 仍在演進，完整實作有鎖定風險
2. 工程師預估完整實作需 2-3 週，短期 ROI 不划算
3. 現有 `AgentMessage` 通道已夠用，先做雙向通訊驗證 PoC
4. Phase 2 再決定是否追加 AgentCard / Task 格式

**分工**：Agent 系統工程師先定義 `AgentMessage` struct + transport adapter trait，不實作 REST Gateway。

---

### TL-DECISION-2026-05-08-002：DuDu Dreaming 優先於 Temporal Memory

**議題**：PM 提案中 P0 有 Dreaming、P1 有 Temporal State Memory，如何排序？

**TL 裁定**：**Dreaming > Temporal Memory**，原因：
1. Anthropic 已上線 Dreaming，是直接競品威脅，防禦優先
2. Temporal Memory（Zep 風格）需要 SQLite schema migration，與 Dreaming Worker 的 memory store 設計高度耦合
3. 建議：Dreaming SDD 時同步設計 `valid_from/valid_until` metadata（為 Temporal Memory 預留介面）

---

### TL-DECISION-2026-05-08-003：Token Cost Defense 執行序列

**TL 裁定**：執行序列維持文件設計：Phase 0 → Phase 2.5 → Phase 3（→ Phase 4 視 V2 API 穩定性決定）。  
**不跳過 Phase 0**，原因：Phase 0 是觀測層，沒有可觀測性就沒有回饋迴路，最佳化沒有依據。
