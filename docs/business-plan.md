> Version: 1.1 | Date: 2026-03-29 | Product Version: v0.8.12

## 一、執行摘要

DuDuClaw 是一套以 Rust 打造的 Claude Code 延伸層（Extension Layer），提供多頻道路由（LINE/Telegram/Discord）、Agent 協作、自主演化引擎、本地推論、成本防禦等企業級功能。本企劃書規劃將 DuDuClaw 從開源專案轉型為可持續營利的商業產品，鎖定三大市場：

1. **軟體產品販售** — 授權 + 一體機 + SaaS
2. **企業導入與教育訓練** — SI 專案 + 認證課程 + 政府標案
3. **傳統產業數位轉型** — 垂直產業解決方案 + 經銷體系

**Year 1 營收目標**：NT$2,100,000–4,000,000
**Year 2 營收目標**：NT$8,000,000–12,000,000
**損益兩平點**：Year 1 Q3（保守估計）

## 二、授權策略與智財保護

### 2.0.1 授權模式：BSL 1.1（Business Source License）

DuDuClaw 採用 **BSL 1.1** 授權，平衡開源社群信任與商業營收保護：

| 使用情境 | 是否需要付費授權 |
|---------|----------------|
| 個人學習、研究、開發 | 免費 |
| 教育機構教學 | 免費 |
| 非營利組織使用 | 免費 |
| 企業內部使用（不對外提供服務） | 免費 |
| **用 DuDuClaw 對外提供商業服務**（客服、SaaS） | 需 Pro / Enterprise 授權 |
| **將 DuDuClaw 嵌入自有產品販售** | 需 Enterprise / OEM 授權 |
| **以 DuDuClaw 為基礎提供託管服務** | 需 Enterprise 授權 |

**Change Date**：每個版本發布 36 個月後自動轉為 MIT，確保長期社群信任。

**選擇 BSL 1.1 的理由**：
- Sentry、CockroachDB、Cline Teams 均採用此模式，市場接受度高
- 個人/教育/非商業完全不受影響 → 社群持續成長
- 商業使用付費 → 保護營收來源
- 36 個月後轉 MIT → 建立「不會被鎖死」的信任感
- 相較 AGPL：對企業更友善（AGPL 常被企業法務部門拒絕）

### 2.0.3 智財保護措施

| 保護層 | 措施 | 說明 |
|--------|------|------|
| **商標** | 「DuDuClaw」+ 爪印 Logo 商標註冊（類別 9, 42） | Fork 不能使用 DuDuClaw 名稱 |
| **授權** | BSL 1.1 + 商業使用條款 | 商業使用需付費授權 |
| **技術門檻** | 12 Rust crate + PyO3 + React 19 | Fork 容易但長期維護極難 |
| **更新速度** | 原作者永遠領先 Fork 1+ 個版本 | 時間差是天然護城河 |
| **安全補丁** | 商業版即時、開源版延遲 30 天 | 企業客戶需要即時安全更新 |
| **認證體系** | DCP / DCA 認證只有官方能發 | 品牌壟斷 |
| **經銷網路** | 在地信任關係不可 Fork | 人際網路無法複製 |
| **數據飛輪** | Memory + Evolution 越用越精準 | 遷移成本隨時間增加 |

## 三、產品定位與競爭優勢

### 3.1 產品定位

「企業級 AI Agent 基礎設施 — 一套系統取代客服軟體 + CRM + 自動化工具」

### 3.2 核心競爭優勢

| 優勢 | 說明 | 競品比較 |
|------|------|---------|
| **資料主權** | 100% 本地部署，資料不離開企業 | 多數 SaaS 需上傳至雲端 |
| **零 API 成本** | Evolution Engine ~90% 對話免 LLM 呼叫 | 競品每次對話都消耗 API |
| **單一執行檔** | Rust 編譯，<100MB，跨平台 | 多數需 Docker + 多服務部署 |
| **多頻道原生** | LINE + Telegram + Discord 內建 | 多數僅支援 1-2 個頻道 |
| **ERP 整合** | Odoo 15 個 MCP 工具，零改造 | 需額外開發或購買中介軟體 |
| **自主演化** | SOUL.md 版本控制 + 24h 自動回滾 | 多數僅靜態提示詞 |
| **成本防禦** | 多帳號輪替 + 本地推論 + Token 壓縮 | 多數無成本控制機制 |
| **繁中優先** | CJK Token 估算、zh-TW UI | 多數以英語市場為主 |

### 3.3 目標客戶畫像

| 客群 | 特徵 | 痛點 | 預算範圍 |
|------|------|------|---------|
| **中小企業主** | 年營收 NT$1,000-5,000 萬，5-50 人 | LINE 客服人力成本高、回覆慢 | NT$3,000-10,000/月 |
| **中型企業 IT** | 年營收 NT$5,000 萬-5 億，有 IT 部門 | 多系統整合困難、AI 導入無從下手 | NT$300,000-600,000 專案 |
| **傳產製造業** | 年營收 NT$1-10 億，工廠 + 業務 | SOP 查詢不便、異常通報慢 | NT$10,000-30,000/月 |
| **貿易/批發業** | 年營收 NT$5,000 萬-3 億 | 報價回覆慢、帳款追蹤漏洞 | NT$5,000-15,000/月 |

## 六、風險評估與緩解

| 風險 | 可能性 | 影響 | 緩解措施 |
|------|--------|------|---------|
| Claude API 大幅漲價 | 中 | 高 | 本地推論已備、LeastCost 策略、Token 壓縮 |
| 競品（OpenClaw）推出類似功能 | 高 | 中 | 深耕台灣在地化 + 垂直產業 + 服務差異化 |
| **被 Fork 複製/轉賣** | **中** | **中** | **BSL 1.1 授權限制商業使用 + 商標註冊 + 閉源商業模組 + 安全補丁時差 30 天** |
| **推廣內容太詳細導致 DIY** | **低** | **低** | **內容分層策略（§2.0.4）：公開教 What/Why，付費教 How Well。會自裝的人本來就不是付費客戶** |
| 傳產接受度低 | 中 | 中 | 體驗營降低認知門檻、政府補助降低成本門檻 |
| 單人團隊產能瓶頸 | 高 | 高 | M3 招兼職、M6 招全職、M8 啟動經銷商 |
| 法規風險（AI 相關法規） | 低 | 高 | 資料不離境、Container 隔離、稽核日誌完備 |
| 客戶流失率高 | 中 | 中 | Evolution 持續優化 → 越用越好、維護合約綁定、數據飛輪提高遷移成本 |
| **中國市場抄襲** | **高** | **低** | **主戰場在台灣，在地信任關係不可複製。BSL 1.1 提供法律基礎** |

### 6.1 護城河分析

真正的護城河不是程式碼，而是以下不可複製的資產：

| 護城河 | 可被 Fork？ | 說明 |
|--------|-----------|------|
| 程式碼 | 可 | 但 12 crate + PyO3 + React 19 長期維護極難 |
| DuDuClaw 品牌 | 不可 | 商標註冊保護 |
| 產業 SOUL.md 調校經驗 | 不可 | 需數百次真實對話迭代 |
| 客戶成功案例 | 不可 | 需時間累積 |
| 經銷商網路 | 不可 | 在地人際信任 |
| DCP/DCA 認證體系 | 不可 | 品牌壟斷 |
| 社群（Discord/LINE） | 不可 | 人跟著人走 |
| Anthropic Partner 關係 | 不可 | 官方認證 |
| 更新速度 | 不可 | 原作者永遠領先 |
| 數據飛輪 | 不可 | Memory + Evolution 越用越準，遷移成本高 |
