## 🏗️ 技術決策記錄

### TL-DEC-2026-05-26-001：BLK-7 PyO3 兼容性修復策略

**背景**：系統升級至 Python 3.14，PyO3 0.24.2 上限為 3.13，導致 `duduclaw-bridge` crate build 失敗，影響整個 workspace CI。

**決策**：
1. **短期（今日）**：基礎設施工程師在 `.cargo/config.toml` 或 CI 環境設定 `[env] PYO3_USE_ABI3_FORWARD_COMPATIBILITY = "1"`，利用穩定 ABI 建構，消除 build 失敗。
2. **長期（W25 內）**：評估 PyO3 ≥ 0.25 是否已正式支援 Python 3.14；若有，升級 `Cargo.toml` 中的 `pyo3` dependency。升級前需確認 API 破壞性變更（特別是 `PyResult`、GIL API 在 PyO3 ≥ 0.22 的遷移路徑）。
3. **不採用**：降級 Python 至 3.13（會影響開發環境一致性，且 3.14 是長期趨勢）。

**向下兼容性**：選項 A 使用 stable ABI，跨 Python 版本兼容性更強，不破壞現有 Python 邏輯。
