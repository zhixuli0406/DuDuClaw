### TL-DEC-2026-05-16-001：3 個 Active Worktrees 建議全數棄置

**背景**：TL 確認三個 worktree 狀態：
- `worktree-agent-a518d165` — HEAD = `deeeced`（v0.9.7）
- `worktree-agent-a106220c` — HEAD = `deeeced`（v0.9.7）
- `worktree-agent-afa20d3a` — HEAD = `deeeced`（v0.9.7）

三個 worktree 均指向相同 commit，與 `main`（v1.14.0，4b13db5）差距 **50+ 版本**（v0.9.7 → v1.14.0），且 diff vs main = 0 uncommitted changes，代表這些 worktree 是空殼，所有工作要嘛已合入 main，要嘛從未啟動。

**決策**：建議全數執行 `git worktree remove`，清理 `.claude/worktrees/`，釋放資源，下次需要 isolation 時重新建立。

**影響**：無資料損失風險，`main` branch 不受影響。

### TL-DEC-2026-05-16-003：PM 提案評估 — Typed Memory Layer（提案 2）

**背景**：PM 建議引入 13 類型語意記憶分類（參考 Memanto 論文 arXiv 2604.22085），問架構方向：疊加 vs 底層重構。

**TL 裁定**：採用**疊加方案**（Layered Approach）。
- 原因：底層重構（~1 week）風險高，且 v1.14.0 Redaction 剛完成，避免記憶系統雙軌重構同時進行。疊加方案（2-3 days）可立即提升查詢精準度，且為漸進式，可在下個 sprint 逐步轉移至原生支援。
- 實作目標：在現有記憶層加入 `memory_type` enum field（13 類型 subset，先支援 8 類），type-filtered retrieval API，衝突偵測 hook。

### TL-DEC-2026-05-16-004：PM 提案評估 — Orchestration RL Spike（提案 3）

**TL 裁定**：**授權 0.5 sprint PoC**（3 天 timeboxed）。

- 原因：arXiv 2605.02801 的 stopping decision 空白是學術前沿，DuDuClaw 已有 orchestration trace 資料（channel_reply、session、GVU 日誌），具備天然數據集。若 PoC 結果有效，可成為技術護城河。
- 邊界：不進入生產，只做「trace → stopping classifier 可行性評估」。
- **需要 Agnes 確認**：AI 工程師（Python 端）可投入此 PoC 嗎？若需動 Rust 端 trace 格式，需 TL 協調基礎設施工程師。
